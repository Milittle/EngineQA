# 场景 09：时段投放配置错误

- index: ad_opt_scene_09
- id: dayparting_misconfig
- tags: 时段投放, 流量窗口, 配置错误

## 场景描述
业务要求覆盖全天，但投放仅在凌晨有少量流量，白天基本不出量。

## 关键观测
- 流量集中在少数小时段。
- 白天 `request` 正常但计划无投放响应。
- 时段系数中存在 0 值配置。

## 常见根因
- 时区配置错误，导致投放窗口整体偏移。
- 时段模板引用错误，加载了测试模板。
- 自动规则将低效时段全部关停且未恢复。

## 排查步骤
1. 核验账户时区、系统时区、报表时区一致性。
2. 检查时段模板 ID 与期望模板是否匹配。
3. 查看自动规则执行日志，确认是否误触发。
4. 以小时粒度回放投放决策链路。

## 处置建议
- 统一时区定义并锁定发布校验。
- 将时段模板纳入变更审批与回滚机制。
- 自动规则增加“最小在线时段”保护。

## 回归标准
- 白天核心时段恢复稳定投放。
- 小时级消耗曲线符合业务预期。
